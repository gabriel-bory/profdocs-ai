# Assistant Evolution Roadmap

This roadmap explains how ProfDocs AI can evolve toward context-aware assistant patterns without overclaiming current capabilities.

The current product is **ProfDocs AI**, a professional document intelligence platform with Context Engine architecture studies.

The universal assistant vision remains a **research track**.

## Current foundation

ProfDocs AI currently includes:

- Angular 21 web shell,
- NestJS API foundation,
- Astro/Starlight docs,
- PostgreSQL 18 local infrastructure,
- GitHub Actions,
- GitHub Pages,
- Playwright visual evidence,
- User Manual,
- DOCX/PDF manual export,
- Context Engine architecture study,
- Context Engine UI study,
- ADRs for provider strategy, Human Approval Gate and prompt injection boundaries.

## Next foundation

The next architecture layer should document:

```text
Project Stack Detector
  -> Local Skill Registry
  -> Skill Manifest
  -> Skill Candidate Selector
  -> Skill Safety Policy
  -> Prompt Orchestrator
  -> Human Approval Gate
  -> Mock Provider
  -> Feedback Memory mock
```

This layer remains documentation and contracts first.

## 7-day roadmap

Focus:

- finalize Skill Registry architecture docs,
- document Stack Detector,
- document Skill Safety Policy,
- document Prompt Orchestrator boundaries,
- document Cloud Free-Tier Strategy,
- keep Universal Assistant as research.

No code implementation required yet.

## 30-day roadmap

Possible next step:

- TypeScript contracts,
- local manifest fixtures,
- mock skill selector tests,
- mock provider tests,
- safe prompt orchestrator contracts,
- documentation updates.

Still no real AI provider required.

## 90-day roadmap

Possible next step:

- rule-based stack detector,
- local skill registry service,
- skill candidate selector,
- feedback memory mock,
- safe repo context retriever,
- Playwright evidence for new UI if implemented.

## Future roadmap

Future product capabilities may include:

- provider abstraction,
- optional local/free provider experiments,
- embeddings,
- PostgreSQL + pgvector,
- source-grounded RAG,
- document ingestion,
- production-grade audit trail.

## Research track

The following ideas remain research-only:

- Ollama deployment,
- CrewAI,
- ChromaDB,
- Neo4j,
- ephemeral containers,
- local desktop agent,
- operating system automation,
- universal assistant multi-domain workflows.

## Why keep research separate

The universal assistant vision is valuable, but implementing it too early would create:

- high complexity,
- security risk,
- unclear product scope,
- infrastructure burden,
- overclaiming,
- maintainability problems.

ProfDocs AI should first become a credible, documented, secure document intelligence platform with a Context Engine foundation.

## Success criteria

This roadmap is successful if:

- current capabilities are described honestly,
- research ideas are not presented as implemented,
- next implementation steps are small and testable,
- the architecture remains free-first,
- the GitHub repository remains professional and credible.
