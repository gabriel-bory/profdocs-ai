This page documents the first visual study for the future **ProfDocs AI Context Engine**.

The purpose of this study is to explore how ProfDocs AI could present prompt orchestration, skill routing, context safety, human approval, provider strategy, and free-tier deployment boundaries in a professional product interface.

> **Current status:** this is a future UI concept. ProfDocs AI does not yet implement real AI/RAG, embeddings, pgvector, Ollama, GitHub Models, CrewAI, autonomous agents, local desktop automation, or automatic skill execution.

## Design goals

- Keep continuity with the existing ProfDocs AI product shell.
- Use documentation-friendly screenshots for GitHub portfolio evidence.
- Separate current foundation, future product direction, and research-only capabilities.
- Make Mock Provider, Human Approval Gate, and no autonomous execution visible.
- Provide narrow responsive references down to 250px.
