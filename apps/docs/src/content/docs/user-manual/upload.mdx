The Upload section presents the planned document ingestion workflow for ProfDocs AI. It shows where users will eventually add documents to the workspace so they can be validated, processed, and prepared for future AI-assisted review.

In the current MVP, this screen is a placeholder. Its purpose is to communicate the intended upload experience and validate the frontend layout before real backend ingestion is implemented.

## What you can do here

Use the Upload section to understand the planned document ingestion experience:

- Identify where users will start a future document upload workflow.
- Understand the intended role of file validation and processing.
- Preview how the upload area will fit into the document intelligence workspace.
- Review the product direction for ingestion, metadata extraction, and processing status.
- Validate the visual foundation before connecting real backend services.

## How to use this screen

Open the web application and navigate to the Upload section from the main application navigation.

The current screen does not upload files yet. It is a visual placeholder that documents the future workflow and keeps the user manual aligned with the current MVP state.

## Current MVP status

The current Upload section includes:

- A professional upload workflow placeholder.
- A clear area for future document ingestion.
- Static explanatory content.
- Playwright-generated visual evidence for documentation.

The current MVP does not yet include real file selection, backend upload, validation rules, virus scanning, metadata extraction, storage, or ingestion status tracking.

## Planned improvements

Future versions of the Upload section are expected to include:

- File selection and drag-and-drop upload.
- Supported file type validation.
- Upload progress feedback.
- Backend ingestion with NestJS.
- Document storage and metadata persistence.
- Processing status and error messages.
- Integration with the document library.
- Preparation of uploaded documents for grounded AI/RAG workflows.
