The dashboard is the main overview screen of the ProfDocs AI web application. It gives users a quick view of the current document intelligence workspace and introduces the main areas of the MVP shell.

The dashboard is generated from the real Angular application and documented with Playwright visual evidence.

![ProfDocs AI dashboard overview in light mode](/evidence/manual/dashboard-desktop-light.png)

## What you can do here

Use the dashboard to:

- Review the current workspace overview.
- Understand the main document intelligence areas available in the MVP.
- Access the document library, upload placeholder, and AI preview sections.
- Confirm that the application shell is responsive and ready for future workflows.
- Switch between light and dark visual modes.

## How to use this screen

Open the web application and start from the dashboard. From here, you can review the workspace summary and move to the main sections using the application navigation.

The current dashboard is designed as a visual and product foundation. It does not yet display live backend metrics or real document processing data.

## Dark mode

ProfDocs AI supports a dark visual mode. This helps validate the design system and provides a foundation for accessible theme switching in future versions.

![ProfDocs AI dashboard overview in dark mode](/evidence/manual/dashboard-desktop-dark.png)

## Mobile layout

The dashboard adapts to smaller screens. The responsive layout is part of the current frontend foundation and is validated with Playwright-generated evidence.

![ProfDocs AI dashboard overview on a mobile viewport in light mode](/evidence/manual/dashboard-mobile-light.png)

The mobile layout is also validated in dark mode.

![ProfDocs AI dashboard overview on a mobile viewport in dark mode](/evidence/manual/dashboard-mobile-dark.png)

## Current MVP status

The dashboard currently includes:

- A professional application shell.
- Workspace overview content.
- Responsive desktop and mobile layouts.
- Light and dark theme support.
- Static MVP information used to communicate the intended product direction.

The dashboard does not yet include live analytics, authenticated user-specific data, real document counts, backend status, or AI-generated insights.

## Planned improvements

Future versions of the dashboard are expected to include:

- Real document processing metrics.
- User-specific workspace information.
- Upload and ingestion status.
- Recent document activity from the backend.
- AI/RAG readiness indicators.
- Notifications for failed or completed document processing jobs.
