The AI preview section presents the intended direction for grounded document intelligence in ProfDocs AI. It shows how users may eventually interact with document-based AI features after documents are uploaded, processed, and made available for retrieval.

In the current MVP, this screen is a placeholder. Its purpose is to communicate the future AI-assisted review experience while keeping the manual honest about the current implementation state.

![ProfDocs AI grounded RAG chat placeholder in light mode](/evidence/manual/ai-preview-desktop-light.png)

## What you can do here

Use the AI preview section to understand the planned AI-assisted experience:

- See where future grounded document questions may be asked.
- Understand the intended role of retrieval-augmented generation.
- Preview how AI responses may be presented inside the workspace.
- Understand that the AI area depends on future upload, ingestion, and retrieval features.
- Validate the product direction before connecting a real AI backend.

## How to use this screen

Open the web application and navigate to the AI Preview section from the main application navigation.

The current screen does not send prompts to an AI model yet. It documents the planned interaction area and shows how grounded AI support may fit into the future document review workflow.

## Dark mode

The AI preview placeholder is also documented in dark mode to confirm that the future chat area remains readable and visually aligned with the dark application theme.

![ProfDocs AI grounded RAG chat placeholder in dark mode](/evidence/manual/ai-preview-desktop-dark.png)

## Current MVP status

The current AI preview section includes:

- A professional placeholder for future AI-assisted document review.
- Static explanatory content.
- A visual foundation for a future grounded RAG workflow.
- Playwright-generated visual evidence for documentation.

The current MVP does not yet include real AI chat, vector search, document retrieval, citation grounding, prompt history, model configuration, or backend AI orchestration.

## Planned improvements

Future versions of this section are expected to include:

- Grounded question answering over uploaded documents.
- Retrieval-augmented generation using indexed document content.
- Source references and citations from documents.
- Conversation history.
- AI response validation patterns.
- Backend integration with NestJS services.
- Future pgvector-based retrieval support.
- Clear user feedback when an answer cannot be grounded in available documents.
