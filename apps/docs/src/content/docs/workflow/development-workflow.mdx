ProfDocs AI follows a professional workflow from the first MVP.

## Branches

```text
main
  └── setup/fullstack-foundation
```

## Commit Convention

The project follows Conventional Commits:

```text
<type>(optional-scope): <description>
```

Examples:

```text
chore: initialize full-stack monorepo apps
docs: add Astro Starlight documentation site
feat(api): add health check endpoint
```

## Pull Requests

Every relevant change should go through a Pull Request, even when working alone.
